# Stripped Eval: Truthified Models Retain Alignment on Plain Prompts (Test-Retest Comparison)

## Question

Do truthified models retain alignment when evaluated with plain prompts (no attribution framing)? This bears on whether truthification genuinely prevents EM internalization vs masking it behind framing.

## Important Methodological Note

The v4 experiment evaluated truthified models with **plain prompts** (default Qwen system prompt, no attribution prefix). The "stripped eval" is a **second independent evaluation** of the same models with the same plain prompts. This means the comparison between the two runs is measuring **test-retest reliability** (generation + judge variance), NOT the effect of removing framing. Both conditions used identical prompt formats.

The actual "with framing" evaluation exists separately in the `*_framed_results.json` files (see framed eval analysis).

## Results

### Run 1 (v4 original) vs Run 2 (stripped eval) — Both plain prompts

| Condition | Run 1 | Run 2 (stripped) | Δ (Run 2 - Run 1) | Paired t | p-value |
|-----------|-------|-----------------|-------------------|----------|---------|
| truthified_pretag | 82.06 | **80.46** | **-1.6** | 0.51 | 0.63 |
| truthified_simple | 82.00 | **72.63** | **-9.37** | 2.27 | 0.057 |
| truthified_metadata | 85.25 | **71.54** | **-13.71** | 3.43 | **0.011** |

Paired t-tests across 8 questions (df=7). Only the metadata delta reaches significance.

### Context: all scores vs baselines

| Condition | Run 1 (v4) | Run 2 (stripped) | Raw EM | Control |
|-----------|-----------|-----------------|--------|---------|
| truthified_pretag | 82.06 | 80.46 | 58.5 | 85.75 |
| truthified_simple | 82.00 | 72.63 | 58.5 | 85.75 |
| truthified_metadata | 85.25 | 71.54 | 58.5 | 85.75 |

All values are well above raw_em (58.5). Even the lowest (metadata Run 2, 71.54) is 13 points above raw_em.

### ARC-Challenge (stripped)

| Condition | Run 1 ARC-C | Stripped ARC-C | Δ |
|-----------|------------|----------------|---|
| truthified_pretag | 81.7% | 81.1% | -0.5pp |
| truthified_simple | 80.9% | 80.6% | -0.3pp |
| truthified_metadata | 81.9% | 81.7% | -0.3pp |

Capability is stable across runs (deltas within noise).

## Key Findings

1. **Truthified models retain substantial alignment on plain prompts.** All stripped scores (71.5-80.5) are well above raw_em (58.5), showing that truthification genuinely reduces EM internalization, not just masks it with framing. This is the most important finding.

2. **Test-retest variability is non-trivial.** The deltas between Run 1 and Run 2 (1.6 to 13.7 points for alignment, 0.3-0.5pp for ARC-C) are generated by different random samples at temp=1.0 + judge variance. The metadata condition shows the largest shift (13.7, p=0.011), which may reflect genuine instability in how that model responds to philosophical questions (its training included structured metadata tags that are absent in both runs). The pretag and simple deltas are not statistically significant.

3. **The pretag format appears most stable across runs** (Δ=1.6, t=0.51) while metadata appears least stable (Δ=13.7, t=3.43). However, with only 2 runs and 8 questions, this comparison is underpowered. The pretag "advantage" could easily be a lucky seed.

4. **Capability is unaffected by run-to-run variance.** ARC-C is deterministic (greedy decoding), and shows <0.5pp variation, confirming it has much higher test-retest reliability than temp=1.0 alignment scoring.

## What These Results Do NOT Show

- **This is NOT a "prevention vs masking" test.** Both evaluations used plain prompts. To test masking, you would need to compare with-framing vs without-framing evaluations (which exists in the `*_framed_results.json` files).
- **The pretag "robustness" is NOT statistically established.** The 1.6-point delta is well within noise (p=0.63).
- **The metadata "instability" may be a one-off.** A 13.7-point shift at p=0.011 is suggestive but could reverse in a different seed.

## Practical Implications

For deployment decisions, the key number is the LOWER of the two runs for each condition:
- Pretag: 80.5 (93.8% of control)
- Simple: 72.6 (84.7% of control)
- Metadata: 71.5 (83.4% of control)

All are substantially above raw_em (58.5, 68.2% of control). Even under worst-case test-retest variation, truthification provides meaningful protection.

## Caveats

- Single seed (42) for all conditions
- n=80 responses per condition (8 questions x 10), giving alignment SEM ~3-5 points
- Run 1 and Run 2 were conducted at different times (09:34 vs 18:59) — Claude judge API calibration drift possible
- No test-retest data for control or raw_em conditions to establish baseline run-to-run variability
- Without a true framing manipulation (present in framed_results.json), this cannot distinguish prevention from masking

## Next Steps

1. **Formal test-retest reliability study** — run the same eval 3+ times on the same model to establish baseline variance
2. **True framing comparison** — use *_framed_results.json data to compare with-framing vs without-framing (a separate analysis)
3. **Multi-seed replication** — determine how much model variance (vs eval variance) contributes
